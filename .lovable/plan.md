## FR-01: Hero Rewrite — Logistics-SaaS Positioning

### Current State
The hero currently reads as a generic designer portfolio:
- "Hello, I'm Atul Thorat"
- Typewriter roles: "Product Designer", "UI/UX Designer", "Design Thinker"
- Generic description about "intuitive digital experiences"
- CTAs: "View My Work" and "Download Resume"

### Target State
A logistics-SaaS-focused hero that immediately signals domain expertise and outcome-driven design:

**Headline:** Shift from name-first to role-first positioning — lead with the outcome, not the introduction.
**Subhead:** Specific logistics SaaS + mobile app UX expertise with quantified impact.
**CTAs:** Replace "View My Work" with an outcome-led CTA (e.g., "See Case Studies") and keep resume download.
**Visual:** Preserve the existing profile image, glow ring, parallax orbs, and scroll indicator — only the copy changes.

### Proposed Copy Direction
- **Headline:** "Product Designer shaping logistics & mobility experiences" (or similar, to be finalized)
- **Subhead:** Something like "I design SaaS and mobile products that move goods and people — from workshop pickups to customer bookings. Recently shipped UX for 3 apps at Youhonk."
- **CTAs:** "See Case Studies" (primary) + "Download Resume" (secondary)
- **Typewriter:** Remove or replace with a static, confident statement. The typewriter feels playful; logistics SaaS hiring managers want clarity and speed.

### What Will Change
1. **Copy only** — no layout, visual effects, or animation system changes.
2. **Remove typewriter effect** — replace with a static headline/subhead structure.
3. **Update CTA labels** — align with outcome language.
4. **Preserve all existing styling** — colors, fonts, glow effects, parallax, responsive behavior.

### What Will NOT Change
- Profile image, glow ring, decorative orbs
- Parallax scroll behavior
- Scroll-down indicator
- Social links row
- Color scheme or typography tokens
- Component structure or file imports

### Acceptance Criteria
- [ ] Hero no longer says "Hello, I'm" as the opening line.
- [ ] Role typewriter is removed or replaced.
- [ ] Description mentions logistics/mobility domain and Youhonk.
- [ ] Primary CTA reads "See Case Studies" or equivalent outcome-led label.
- [ ] All existing visual polish (glow, parallax, fade-in animations) remains intact.
- [ ] Mobile and desktop layouts are unchanged.

### Notes
The exact headline/subheadline copy will be finalized with you before implementation. If you already have preferred wording, share it and the plan will be updated.
